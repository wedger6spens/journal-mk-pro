# journal-mk-pro

一个用于归档和发布独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放和发布多个独立的 HTML 页面。这些页面以静态文件的形式存在，不依赖于特定域名或后端服务，方便直接通过 GitHub Pages 或其他静态托管服务进行访问。

仓库本身不限定页面内容的具体主题或用途，仅作为页面文件的存储与分发载体。

## 目录说明

```
.
├── index.html          # 首页 / 入口页面
├── pages/              # 存放各独立 HTML 页面的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：仓库的默认入口页面，通常用于导航或展示页面列表。
- `pages/`：所有独立 HTML 页面的存放位置。每个页面可自包含样式与脚本，也可引用 `assets/` 下的公共资源。
- `assets/`：用于存放页面共享的静态资源文件，按类型分目录组织。

## 页面归档说明

- 每个 HTML 页面在 `pages/` 目录下独立存在，文件名即为页面标识。
- 页面之间无强制依赖关系，可单独访问或更新。
- 如需新增页面，直接在 `pages/` 下创建新的 HTML 文件，并在 `index.html` 中添加链接即可。
